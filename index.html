    • Open Weather Analysis: Latitude



    This site visualizes analysis from Homework 6: Python APIs. The dataset was pulled from the OpenWeather API for a randomized list of over 500 cities worldwide, and then plotted and analyzed. The homework called for analysis of latitude versus a number of attributes brought back in the API responses. See the below visualizations to learn more.

    The Openweather API is able to take a combination of latitude and longitude data, and provide back a response for the weather in that given area. The API has the ability to return data in both XML and JSON formats, and also has the ability to provide resulting datasets with varying units of measurement (standard, metric, imperial). Once the API is called, a response is generated which includes meterological conditions at the date/time of the request. If data is available for the given latitude/longitude combo, specific attributes of the response data are saved back into lists which are then merged into a Pandas Dataframe. If no data is available for a given latitude/longitude combo, the response is skipped. Once the full list of data was aggregated, then individual plots were generated based on instructions provided in the homework requirements.
